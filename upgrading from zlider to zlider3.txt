Analysis of Current vs. Proposed Model:
Current State:

Each presentation is a separate .zlides file
User must navigate filesystem to open/save
No quick switching between different presentation contexts
Scattered files make organization difficult

New workspace Model Benefits:

Single source of truth: One workspace.json file
Visual organization: Folders visible in sidebar
Quick switching: One click to change presentations
Auto-save: No manual save needed
Better UX: More like Spotify playlists than Word documents

Proposed Architecture:
pythonworkspace.json structure:
{
  "folders": [
    {
      "name": "Seminars",
      "presentations": [
        {
          "name": "Intro to AI",
          "zlides": [...],
          "settings": {...}
        }
      ]
    }
  ],
  "recent": ["Intro to AI", "Daily Standup"],
  "settings": {
    "dark_mode": true,
    "auto_close_mode": false
  }
}
